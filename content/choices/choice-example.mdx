# Choice Anatomy

Choices are how contracts evolve in DAML.

## Choice Structure

```daml
    choice ChoiceName: ReturnType
      with
        inputField: InputType
      controller exercisingParty
      do
        -- business logic here
        return result
```

## Types of Choices

### Consuming Choices

By default, choices **consume** the contract:

```daml
    choice Consume: ()
      controller owner
      do
        archive self
```

### Non-consuming Choices

Use `consuming` or `nonconsuming` keywords:

```daml
    nonconsuming choice Peek: Decimal
      controller owner
      do
        return amount
```

## Choice Guards

Validate preconditions before executing:

```daml
    choice Transfer: ContractId Asset
      with
        newOwner: Party
      controller owner
      do
        assert (amount > 0.0)
        create this with owner = newOwner
```

## Multiple Controllers

Choices can have multiple controllers:

```daml
    choice JointTransfer: ContractId Asset
      with
        newOwner: Party
      controller issuer, owner
      do
        create this with owner = newOwner
```
