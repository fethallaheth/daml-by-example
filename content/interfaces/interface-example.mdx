# Interfaces

Interfaces define a set of choices that multiple templates can implement.

## Defining an Interface

```daml title="TokenInterface.daml"
daml 2.0
module TokenInterface

interface Token
  where
    getOwner: Party
    getAmount: Decimal

    choice Transfer: ContractId Token
      with
        newOwner: Party
      controller getOwner self
      do
        return self
```

## Implementing an Interface

```daml title="FungibleToken.daml"
daml 2.0
module FungibleToken

import TokenInterface

template FungibleToken
  with
    issuer: Party
    owner: Party
    amount: Decimal
  where
    signatory issuer, owner
    interface instance Token for FungibleToken where
      getOwner = getField @"owner" self
      getAmount = getField @"amount" self

      choice Transfer: ContractId Token
        with
          newOwner: Party
        controller getOwner self
        do
          create this with owner = newOwner
```

## Benefits of Interfaces

Interfaces enable:

- **Polymorphism**: Write code that works across multiple template types
- **Composability**: Build complex workflows from simple interfaces
- **Abstraction**: Hide implementation details behind a clean API
